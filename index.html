Capabilities
Everything you need to govern AI spend.
Six pillars of token intelligence — from real-time telemetry to compliance-ready governance.

Real-Time Telemetry

Every API call captured in milliseconds. Scout connectors intercept at the fetch level — zero code changes, zero SDK lock-in. Works with OpenAI, Anthropic, Ollama, and 10+ providers.

Zero Content Principle

HIVE never reads prompts, completions, or API keys. Metadata only. GovernanceBlock fields pii_asserted and content_asserted are structurally frozen to false.

Intelligence Engine

Cost modelling, anomaly detection, spend forecasting, behavioural clustering, and department fingerprinting. Turn consumption data into actionable insights.

Shadow AI Detection

Unsanctioned AI providers are a governance risk. HIVE detects, quantifies, and surfaces shadow AI usage in real-time. Know what you don't know.

Department Analytics

Tag every call by department, project, and environment. See who spends what, where waste lives, and which teams get the most value from AI.

Glassmorphic Dashboard

A stunning dark-theme dashboard that makes governance visible, not buried. KPIs, charts, real-time feeds, and an AI chat assistant — all in one pane of glass.

Quick Start
Running in 60 seconds.
One command to start the full stack. Docker or bare metal — your choice.
Docker
# Clone and start everything
git clone https://github.com/vishalm/hiveHQ.git
cd hiveHQ
docker compose --profile cpu --env-file .env.docker up --build -d

# Open the dashboard
open http://localhost:3001
Bare Metal
# Or run without Docker
chmod +x run.sh
./run.sh --seed 200

# Dashboard at :3001 · API at :3000 · Docs at :3002 · Swagger at :3000/api/docs
Governance
Compliance is structural, not optional.
Every event carries a GovernanceBlock. Privacy guarantees are frozen at the schema level.
Frozen

Zero Content

pii_asserted: false and content_asserted: false are z.literal(false) in the schema. They cannot be set to true. Ever.

Enforced

Data Residency

Node region validation rejects events that violate residency policy. Federated and open deployments enforce geographic boundaries.

Cryptographic

Ed25519 Signatures

Batches are signed with Ed25519 keys. Audit chain with per-event hashing and Merkle roots. Tamper-evident by design.

Policy Engine

Admission Control

Configurable policy rules with priority ordering. Deny events by provider, department, region, or any metadata field.
